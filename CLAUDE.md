# Long Tales — Project Guide

This file is the standing brief for every session I run in this project. Read it first, every time.

## Session start protocol

At the beginning of every session, in this order:

1. Read `About Me/personal-context.md`. Who Saurabh is, how he works, what he expects from outputs.
2. Read `About Me/anti-ai.md`. Writing rules. These are strict, not suggestions.
3. Read `About Me/brand-context.md`. Brand identity, voice, positioning, what Long Tales is and is not.
4. Read `About Me/failures.md`. Past misses. Don't repeat them.
5. Scan `TODO.md`. Know what's pending so I can surface relevant items when context matches.

Only after these five steps do I engage with the new request.

## Behaviour rules across sessions

Never change a file without explicit permission. This applies to every file in the project, including this one. If I believe a change is needed, I propose the diff and wait for a yes.

If I notice something that belongs in `TODO.md`, I surface it and ask before adding. No silent edits.

If I miss the point in a session, log it to `About Me/failures.md` when Saurabh flags it. One entry per miss.

If anything here conflicts with observed reality (files on disk, latest decisions, new facts), trust reality and flag the conflict openly.

Writing style follows `anti-ai.md` by default, regardless of format (doc, deck, label copy, WhatsApp script, pamphlet, website).

Brand voice follows `brand-context.md`. When in doubt, match the voice before chasing the brief.

## Folder map

`About Me/` — meta about Saurabh, the brand, writing rules, and failure log. Lives longest, changes rarely.

`Collaterals/` — approved brand assets only. Packaging, labels, pamphlets, landing pages, photos, logos. Nothing goes here until Saurabh has signed it off.

`Strategy/` — business strategy docs. Ops, financial projections, positioning, launch plans, pricing thinking. May grow subfolders over time.

`TODO.md` — running list of pending decisions and actions across the project.

Work-in-progress drafts do not live in the project folder. They live in the working scratch directory until approved.

## Style defaults for this project

Plain prose over bullet stacks, unless the content is genuinely tabular or a list of items of equal weight.

Village name carries weight. Typographically on packaging, rhetorically in prose. Brand name sits below it.

No em-dashes as pauses. Use periods, commas, or parentheses.

No corporate nouns (solutions, offerings, touchpoints, ecosystem). No wellness vocabulary (authentic, pure, premium, holistic, Ayurvedic).

Specific nouns over abstract ones. Jar, not product. Saurabh, not founder. Karauli, not origin.
